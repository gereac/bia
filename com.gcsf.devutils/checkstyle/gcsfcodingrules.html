This set of codingrules presents the checkstyle configuration used with this release
action have to be taken depending on severity level:
  • error - to be analyzed and fixed
  • warning - this may impact future usage - to be fixed if source is modified next time
  • info - reported for usage with other measures only
id description severity checker
HMI-B01 Checks for braces around code blocks error NeedBraces
HMI-B02 Checks for empty blocks - at least text required error EmptyBlock
HMI-B03 Checks for empty blocks - catch blocks without statements info EmptyBlock
HMI-C01 Checks Javadoc comments for class and interface definitions info JavadocType
HMI-C02 Checks the Javadoc of a public method or constructor info JavadocMethod
HMI-C03 Checks that public variables have Javadoc comments info JavadocVariable
HMI-I01 Checks for imports from a set of illegal packages: com.sun.media error IllegalImport
HMI-I02 Checks for imports from a set of illegal packages: java.lang.reflect error IllegalImport
HMI-I03 Checks that there are no import statements that use the * notation error AvoidStarImport
HMI-I04 Checks for redundant import statements error RedundantImport
HMI-I05 Checks for unused import statements error UnusedImports
HMI-L01 Checks correct indentation of Java Code. Use Ctrl-I for correct indentation warning Indentation
HMI-L02 Checks that long constants are defined with an upper ell. e.g '1000L' error UpperEll
HMI-L03 Checks for TODO: comments error TodoComment
HMI-M01 Checks that the order of modifiers conforms to the suggestions in the Java Language specification warning ModifierOrder
HMI-N01 Checks that local final variables follow naming convention: warning LocalFinalVariableName
HMI-N02 Checks that local variables follow naming convention: warning LocalVariableName
HMI-N03 Checks that member variables (non-static fields) follow naming convention: ^my[A-Z][a-zA-Z0-9]*$ warning MemberName
HMI-N04 Checks that static variables (static, non-final fields) follow naming convention: ^our[A-Z][a-zA-Z0-9]*$ warning StaticVariableName
HMI-N05 Checks that constants (static final fields) variables follow naming convention: ^[A-Z][A-Z0-9]*(_[A-Z0-9]+)*$|^our[A-Z][a-zA-Z0-9]*$ warning ConstantName
HMI-N06 Checks that parameter names follow naming convention: ^a[n]{0,1}[A-Z][a-zA-Z0-9]*$|^args$ warning ParameterName
HMI-N07 Checks that method names follow naming convention: warning MethodName
HMI-N08 Checks that package names follow naming convention: ^[a-z]+(\.[a-z_][a-z0-9_]*)*$ warning PackageName
HMI-N10 Checks that class names follow naming convention: warning TypeName
HMI-N11 Checks that interface names follow naming convention: ^I[A-Z][a-zA-Z0-9]*$ warning TypeName
HMI-N12 Checks that abstract class names follow naming convention: ^[A-Z][a-zA-Z0-9]*$ warning AbstractClassName
HMI-P01 Checks that each variable declaration is in its own statement and on its own line error MultipleVariableDeclarations
HMI-P02 According to Code Conventions for the Java Programming Language , the parts of a class or interface declaration should appear in a given order warning DeclarationOrder
HMI-P03 Checks that switch statement has "default" clause error MissingSwitchDefault
HMI-P04 Check that the default is after all the cases in a switch statement error DefaultComesLast
HMI-P05 Checks for fall through in switch statements Finds locations where a case contains Java code - but lacks a break, return, throw or continue statement. The check honores special comments to supress the warning. By default the text "fallthru", "fall through", "fallthrough", "falls through" and "fallsthrough" are recognized (case sensitive). error FallThrough
HMI-P06 Detects empty statements (standalone ;) error EmptyStatement
HMI-P07 Checks that a local variable or a parameter does not shadow a field that is defined in the same class warning HiddenField
HMI-P08 Checks that classes (except abtract one) define a ctor and don't rely on the default one warning MissingCtor
HMI-P09 Check for ensuring that for loop control variables are not modified inside the for block error ModifiedControlVariable
HMI-P10 Ensure a class is has a package declaration error PackageDeclaration
HMI-P11 Restricts the number of return statements to 2 info ReturnCount
HMI-P12 Checks that string literals are not used with == or != error StringLiteralEquality
HMI-P13 Catching java.lang.Exception, java.lang.Error or java.lang.RuntimeException is to be used for error handling only info IllegalCatch
HMI-S01 Checks for lines longer than 160 characters warning LineLength
HMI-S02 Checks for source files longer than 1500 lines warning FileLength
HMI-S03 Checks for methods and constructors longer than 150 lines warning MethodLength
HMI-S04 Checks the number of parameters of a method or constructor greater than 5 warning ParameterNumber
